## Deletion cost (for subsume / replace rows)

- **Row 3 — parser (replace, eventual):** callers = `ledger-render/SKILL.md` (line 70 reads `metrics.json`), `explain/SKILL.md` (line 33 globs `metrics.json` across sessions), `stop.sh` (lines 26–28 invokes parser via `nohup`). Data migration = none on disk (output schema can be preserved); migration cost is in re-validating attribution math against any new source-of-truth. Config surface = none (no env vars, no settings.json keys). Deletion cost is **deferred** — replacement is conditioned on a stable host-product surface existing, which it does not today; until then the parser is the v1 stand-in we ship-with-canary, not the v1 we delete.
- **Row 7 — `workflow-id.txt` prose handoff (replace):** callers = orchestrator step 1 (CLAUDE.md lines 24–31 prose block), `stop.sh` lines 17–34 (reads the file or archives to orphan). Data migration = none; staging/orphan dirs already exist and the structural replacement reuses them. Config surface = none. Blast radius small: one prose block in CLAUDE.md, one branch in `stop.sh`, one orchestrator step. Risk: a structural enforcer that fails loud is *not* AI-blind — design must keep the failure path off the orchestrator's context.
- **Row 11 — convention `2>/dev/null` (replace by `hook-exec.sh`):** callers = three hook scripts × every external command line (~10–15 invocation sites total). Data migration = none. Config surface = `settings.json` `command` values gain `hook-exec.sh` prefix (extend, see row 8). Blast radius: rewrite each hook command, add wrapper, update settings.json hooks block. Convention text in the upgrade entry and any agent prompts that cite `2>/dev/null` discipline must be updated to point at the wrapper.
- **Row 12 — regex pair (replace):** callers = parser internals only; outputs feed `metrics.json` `verdicts` and `loop_count` fields, which are read by `ledger-render` and `explain`. Data migration = **historical metrics may shift** if any pre-existing `metrics.json` files are re-parsed; in practice none exist today (explore §10: zero populated `diagnostics/` folders), so the migration cost is zero in current state. Config surface = none. Risk: tightening can drop true positives if the canonical `verdict:` / `## Loop N` patterns are not enforced upstream in critic and decision-log prose.

## Requires decision (conflicts)

- **Row 10 — undocumented internal layout vs. coupled parser:** the parser depends on `~/.claude/projects/<encoded-cwd>/<claude-uuid>.jsonl` plus `subagents/agent-<id>.jsonl` plus `.meta.json` siblings. Canon (SRE Ch.6 §As Simple as Possible) says integrate against stable surfaces, not layout. Outside-view names host-product layout change as the modal failure (~40–60% qualitative 6-month survival, unsourced). The user has not chosen between:
  1. **Accept the coupling, add a canary** (cheap, keeps v1 shipping, defers the conflict to first failure)
  2. **Defer / disable the parser** until Anthropic publishes a stable hooks-output spec (preserves canon position, retires the catalyst question for now)
  3. **Replace with the public spec** when one exists (correct end-state, not actionable today)

  This is a frame-level decision the panel should not pre-empt. Flagging for synthesis or for the user to call.

## Net change implied by the review

The labels above are weighted toward `extend` on the surface-level primitives the layer **owns** (rows 1, 2, 4, 5, 8, 9) and toward `replace` on the three places the implementation made convention-not-structure choices (rows 7, 11, 12). One genuine `conflict` remains for the user (row 10).

The review is **not** research theater if synthesis acts on the highest-leverage rows. Three actions, ranked:

1. **Resolve row 10 (conflict on undocumented-layout coupling).** This is the modal failure mode named by both outside-view and canon. The cheapest move is option (a) — accept the coupling and add the silent-zero-attribution canary outside-view §9 named — but the user must call it. Without a decision here, every other improvement is decoration on a parser that may zero-out silently in six months.
2. **Replace the convention-only `2>/dev/null` discipline with `hook-exec.sh` (row 11).** Below-base-rate feature; small, bounded change; structurally enforces the load-bearing AI-blindness invariant. Blast radius is contained to three hook scripts and one settings.json block.
3. **Tighten the verdict and loop regexes (row 12) and add a producer/consumer schema-conformance test for `metrics.json`** (implied by row 4's review action). Both are cheap; both eliminate quiet attribution drift; the schema-conformance test catches the gap explore §gaps named (drift between parser-as-producer and ledger-render/explain as consumers).

Action 1 is a frame-level call. Actions 2 and 3 are implementation-level and should land regardless of how 1 is resolved. If synthesis emerges with only "queue follow-up" against rows 7, 11, 12 and no movement on row 10, the panel has the evidence to call research theater per the requirement.md frame bias.

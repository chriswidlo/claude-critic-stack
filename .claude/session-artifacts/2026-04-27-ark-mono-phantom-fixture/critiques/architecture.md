# Architecture-lens critique — ark-mono phantom fixture

**Unit of analysis:** the workflow + agent + artifact structure of this repo (not the ark-mono target the regression scenario describes). The "architecture" under review is the module boundaries between `tests/regression/`, `.claude/session-artifacts/exemplars/`, `.claude/session-artifacts/<session>/`, the cleanup punch-list, and the prose-cited examples in [CLAUDE.md](CLAUDE.md) and [session-artifacts README](.claude/session-artifacts/README.md).

## 1. Weakest structural link

**Item 5b creates a forward dependency from the regression-test artifact to a contract document that does not yet exist, with no module that owns the contract's schema or lifecycle.** The candidate names "exemplar contract" four times across §Position and §How-this-addresses. Nowhere does it say where the contract lives on disk, which directory's `README.md` defines its required fields, who is allowed to edit it after the first run, or whether a future regression test inherits the same contract or authors a new one. This is a protocol surface introduced without a home module.

The concrete failure mode: item 5b is filed; six months later somebody picks it up; the contract gets authored ad-hoc inside the session-artifact directory of *that* session; it dies with the session because nothing in `tests/regression/` or `.claude/session-artifacts/exemplars/` references it; the next regression test repeats the same authoring exercise from scratch. The "exemplar contract" is in the same structural position the empty `exemplars/` directory was in before this session — a claim of discipline with no backing module.

## 2. Invariants at risk (not the ones the candidate already names)

1. **Citation-style mono-culture in canonical docs.** The repo currently has two citation styles: (i) repo-relative Markdown links to real files, and (ii) example slugs presented as if they were real session ids (the phantom this session is fixing). Step 1 of the candidate introduces a *third* style: a placeholder shape (`<YYYY-MM-DD>-<slug>`) presented as illustrative-with-explanatory-note. This is also forbidden by the path-discipline regime — repo-relative Markdown links with meaningful display names are the only sanctioned in-repo citation style. A placeholder shape inside what reads like a citation slot is a new variant. If the invariant is "everything that looks like a path is a path you can click," the candidate breaks it. If the invariant is "nothing in committed artifacts presents a fabricated real-looking slug," the candidate honors it. The candidate does not name which invariant governs and ships the variant that resolves the second while violating the first.

2. **Punch-list items are atomic units of work, not pointers to specs that will be authored later.** The current punch-list reads as discrete bug-rows. Item 5b as proposed is *not* a bug-row; it is a meta-row that says "first, author a spec, then run a test against it, then promote the result." This collapses three sequential decisions into one row and bets that whoever picks it up reads §Contract minimum content in `candidate.md`. The invariant that "a punch-list item is closeable in one session" silently breaks.

3. **The regression-test artifact is the spec; the agent stack is the implementation.** The candidate's §Named-assumption 2 partially names this, but understates it. The amendment proposed ("add an acceptance criterion for the distiller step") inverts the dependency direction: the test now depends on the agent set, because every time the agent set changes the test must be re-amended before it can be run. If the test is the spec, the agent set should depend on the test (changing an agent's role triggers a test review). If the agent set is the spec, the test is documentation and should be regenerated, not amended. The candidate amends without choosing — it lets the test be whichever the operator finds convenient at amendment time. This is the cycle named in §3.

4. **`.claude/session-artifacts/exemplars/` is a prescriptive artifact directory; `.claude/session-artifacts/<session>/` is a historical-record directory.** The frame-challenger named this; the scope-map preserved the immutability norm; the candidate defers it to item 5b. Deferring an invariant question to a future session is fine *if* the deferral is recorded in the directory whose contract is at risk. It is not. The exemplars directory's `.keep` does not gain a "see item 5b for promotion semantics" link.

## 3. Coupling and direction

- **Volatile→stable expected; volatile→volatile observed.** The regression-test artifact is meant to be the *stable* spec. The agent set is the *volatile* implementation. Step 1 of the candidate keeps that direction. Step 2 reverses it: the test acquires a new acceptance criterion *because* the agent set changed (distiller was inserted). Once that precedent is set, every workflow change implies a test amendment, and the test ceases to be a regression test — it becomes a mirror of the current agent set, by definition unable to detect drift.

- **Layering violation between docs and tests.** [CLAUDE.md](CLAUDE.md) and [session-artifacts/README.md](.claude/session-artifacts/README.md) cite an example slug. That slug, under the original option (a), would have been the *output* of the regression test (a promoted exemplar). The doc layer therefore citationally depends on the test-output layer. The placeholder-slug fix in candidate step 1 cuts that dependency cleanly and is the right call. But §Named-ways-this-could-be-wrong already flags the regression — a future author will replace the placeholder with a real-looking date. There is no enforcement (linter, CI grep, agent rule) preventing this. The architectural fix is a rule — "no fabricated slugs in CLAUDE.md or the session-artifacts README; placeholders only, in `<YYYY-MM-DD>-<slug>` shape" — written into the path-discipline regime in CLAUDE.md, not a polite note in the placeholder itself. The candidate ships the polite note.

- **Cycle introduced.** Candidate step 2 makes item 5b depend on (i) the exemplar contract, which depends on (ii) a spec-currency audit that will run during item 5b execution, which depends on (iii) the regression test as currently written, which depends on (iv) the agent set as currently shipped, which the audit may amend, which feeds back into (i). The candidate's own §Named-assumption 2 says "if multiple drifts exist, the test is closer to stale than incomplete, and item 5b becomes a test rewrite rather than a contract-then-run" — i.e., item 5b *can* expand into rewriting its own input. That is a cycle; it should be cut by either (A) doing the spec-currency audit *now* in this session as a separate artifact, or (B) splitting item 5b into 5b-audit and 5b-run with 5b-run gated on 5b-audit's findings.

## 4. Ignored architectural alternatives

1. **Promote a stub exemplar.** Author a minimal exemplar artifact in `.claude/session-artifacts/exemplars/` that exists *solely* to satisfy the directory's promotion contract — a documented "this is a shape exemplar, not an execution exemplar," with the same `<YYYY-MM-DD>-<slug>` placeholder convention. This makes the empty-directory-as-quiet-erosion problem (frame F2) go away without running the regression and without authoring an exemplar contract. The candidate did not consider this because it conflates "exemplar exists" with "exemplar is execution-derived." Architecturally, those are separable.

2. **Move the regression spec into the agent set.** Instead of `tests/regression/ark-mono-connector-routing.md` as a free-floating document, embed each acceptance criterion inside the agent it targets. The "test" becomes a harness that walks the agent set and collects criteria. This eliminates the spec-currency drift problem entirely.

3. **Delete `tests/regression/` and `.claude/session-artifacts/exemplars/` entirely; replace with a single `.claude/exemplars/` directory whose `README.md` is the contract.** The current two-directory split is the source of every coupling problem in this critique.

## 5. Frame-level objection

The frame-challenger surfaced F4 (calibration of the regression-test artifact). The candidate accepted F4 partially. **It missed the deeper frame-level move F4 implies: if the regression test is an unverified hypothesis, the architectural unit at risk is not the test or the exemplar but the *regression-discipline subsystem as a whole* — the contract between `tests/regression/`, `.claude/session-artifacts/exemplars/`, the agent set, and the docs that cite both.** Item 5b proposes to author a contract for *one* exemplar. The architectural question is whether a regression-discipline subsystem exists at all in this repo, or whether what exists is one orphaned test file, one empty allowlisted directory, and two doc citations to a slug that was never produced.

## 6. Verdict

**rework**

What would change the verdict: name an owner module for the exemplar contract (which directory's `README.md` defines its schema), cut the cycle between item 5b's contract and its audit by splitting 5b into 5b-audit (run in this session or the next, before any contract authoring) and 5b-run (gated on 5b-audit), and write the placeholder-slug rule into the path-discipline regime in CLAUDE.md rather than as prose inside the placeholder itself.

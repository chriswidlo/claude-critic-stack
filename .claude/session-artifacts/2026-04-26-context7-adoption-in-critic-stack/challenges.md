# Frame challenges — Context7 adoption

## Frame as given
The orchestrator committed to a cost/benefit frame: "Is the marginal anti-anchoring cost of injecting library docs into this stack greater or less than the marginal value to design questions, given the stack's actual question distribution? If 'sometimes yes,' what is the minimum-discipline-cost integration shape?" The frame privileges discipline preservation over capability accretion and treats non-adoption (shape A) as the default that adoption must beat. The optimization target is *protecting the anti-anchoring mechanism* while leaving a narrow door open for high-value, library-specific queries.

## Alternative frame
**"Context7 is not a retrieval question; it is a corpus-policy question. The stack already has a defined trust hierarchy (curated canon > web > nothing). Context7 is a third tier — *vendor-authored, vendor-hosted, vendor-versioned* — and the real question is whether the stack's trust hierarchy admits a vendor tier at all. If it does, Context7 is one of N implementations and the choice is administrative. If it does not, no integration shape is acceptable, including shape A as currently scoped (because shape A still leaves the door open as 'pending more evidence' rather than closing it on principle)."**

Under this frame, the entire B/C/D/E debate is a category error: it argues *integration mechanics* for a tier that has not been admitted to the trust model. The orchestrator's frame asks "where does the new pipe go"; this frame asks "is the fluid permitted in the building." If the answer is no, all five shapes collapse to "reject and document why" — a stronger and more durable artifact than "do nothing pending more evidence." If the answer is yes, the work is to write a `trust-tiers.md` policy first, and only then pick an adapter. Either way, the orchestrator's frame is asking the second question before the first.

## Condition under which the current frame is wrong
The current frame is wrong if the stack's actual question log shows that >25% of design questions over the last 90 days name a specific library + version (e.g., "should we use LangGraph 0.2 vs Inngest 3.x for X") rather than patterns/tradeoffs/invariants — because at that frequency, "anti-anchoring cost vs. marginal value" stops being the dominant axis and *retrieval coverage* becomes the dominant axis, at which point the discipline-vs-capability framing flips into a freshness-and-coverage framing where Context7 is competing against canon-refresher cadence rather than against canon-librarian's discipline.

## Pre-emptive challenge to the most likely scope-map preservation
The expected preservation: **canon-librarian and Context7 as separate retrieval lanes** (justified by "different trust models, different freshness semantics, separation of concerns"). I challenge this preservation directly.

- **The stated reason ("different trust models") is weak because** the trust model is not a property of the *backend*; it is a property of the *labeling*. A single librarian agent can label retrievals as `tier:curated`, `tier:vendor-live`, `tier:web` without fragmenting into multiple agents. Cockburn C6 (from canon-librarian distillation) explicitly cautions against port proliferation: 2–4 ports, multiple adapters per port. Two retrieval *agents* is the proliferation Cockburn warns against; two *adapters behind one librarian* is the shape he endorses. Preserving canon-librarian as untouched and adding Context7 as a sibling violates the architectural primitive the canon itself names.

- **The stated reason ("separation of concerns") is weak because** the orchestrator already has to reason about which retrieval source to consult. Splitting that reasoning across two agents pushes the disambiguation cost into the orchestrator's prompt — exactly the "ambiguous tool choice" failure mode flagged in the canon-librarian distillation (C2 / Anthropic Effective Context Engineering: "If a human engineer can't definitively say which tool should be used in a given situation, an AI agent can't be expected to do better"). The preservation creates the failure mode the corpus already warned about.

- **The preservation biases the downstream answer toward** shape C (a dedicated `library-docs` subagent) by making it the path of least resistance — canon-librarian stays pristine, new agent gets the new responsibility, no existing agent has to change. This is the *organizational* easy answer, not the *architectural* right one. The right shape, if any adoption happens, is **subsumption**: Context7 becomes a backend behind canon-librarian, the librarian gains a `tier` field on every returned passage, and the contradiction-required rule is enforced *across tiers* (a vendor passage must be cross-checked against a curated passage when one exists). This collapses C and B into a single answer and demotes D to "irrelevant: corpus-refresher operates on a different cadence than per-query retrieval."

The preservation should be replaced. The default `subsume` of the scope-mapper applies; the burden is on whoever wants two retrieval agents to defend why one cannot do the work with adapters and labels.

## Challenge to the classifier label
The classifier picked `extend` with `new` as alternative. The substantive shift would be to `investigation` — the third reading the classifier itself flagged. Under `investigation`, the question is not "which shape" but "what is the question log telling us about the gap Context7 would fill." The downstream frame becomes empirical first (read the last 90 days of design questions, count library-API bottlenecks, then decide) rather than architectural first. This matters because the current frame *assumes the gap exists at non-trivial frequency* and only asks how to fill it. The `investigation` reframe refuses that assumption and demands evidence before any shape is on the table — including shape A, which under `investigation` is not "do nothing" but "do the measurement." The orchestrator's frame absorbed the user's adoption presupposition more than it disclosed; an `investigation` reframe would have rejected the presupposition outright.

## Frame-level objection the orchestrator should carry forward
The candidate must defend why the trust-tier policy question is not prior to the integration-shape question — i.e., why we are choosing pipes before deciding whether the fluid is admitted to the building.

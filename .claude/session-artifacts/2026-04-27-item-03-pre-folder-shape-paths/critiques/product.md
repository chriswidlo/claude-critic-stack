# Product critic — loop 1

**Verdict: rework.**

## User-visible consequence

The operator (the repo's user) issued a five-word ask: "pick up the third critical." Day-of-ship, what they observe is **three commits, one open binary question routed back to them, a frame revision they did not request, and a 5th edit appended to a line previously declared off-limits**. The single most-noticed surface change: the operator opens git log expecting `item 3 ✅ done` and instead finds commit A (a child-of-`c2fe604` framing they didn't ask for), an open thread named "commit B," and a deferred commit C tied to another item's pipeline. The "done" affordance for item 3 is now gated on operator action they had already implicitly delegated.

## Commitments implied

Three load-bearing commitments, all of which become precedent the moment commit A lands:

- **"Cleanup items spawn 1–3 commits and ≥ 1 operator decision."** Item 3 is the first cleanup ticket to exit the new pipeline; its shape will template items 4–12. Future cleanup work inherits the three-commit envelope.
- **"Commit messages may name parent commits as conceptual ancestors."** The "child of `c2fe604`" framing is a new contract about how commit messages relate to each other. Lightweight to write; load-bearing the moment a second commit cites a "parent" and a third does not.
- **"Open questions can live as commit-shaped placeholders (commit B, optional, pending operator)."** This invents a new artifact class — the *deferred commit slot* — that has no current carrier in the repo. It either becomes a pattern or it becomes a stale TODO.

## Migration burden

- The **operator** must answer the strict-vs-permissive policy question to unblock commit B.
- **Item 1's session/successor** inherits commit C — a known re-touch on the same 4 lines.
- **Future cleanup items (4–12)** inherit the precedent that each item may legitimately split into 3 commits + a question.
- **Future readers of the dogfood entry at line 138** get a parenthetical that did not exist before.

The candidate's framing — "commit B is genuinely deferred because it has its own commit envelope" — is honest about *where* the burden lives but not about whether the burden was generated by the workflow itself. **The policy question is real; the *commit-B-shaped* container for it is manufactured by this candidate.**

## Affordances better / worse

**Better:**
- 4 active-surface links resolve correctly.
- Line 138 forwards a future reader to the live file (additive parenthetical).
- Each commit has a clean stop-condition; each is independently revertable.

**Worse / pruned:**
- The affordance "item 3 is done in one commit" is gone. Replaced by a graph of commits with conditional edges.
- "Plan = ship" coupling is loosened. The operator's plan-then-execute mental model now has a third stage: plan, *re-plan via pipeline*, ship.
- "Commit messages are about what the commit does" is loosened to "commit messages may also re-narrate prior migrations."
- "Done-when conditions are stop-tests" is loosened — Done-when #5 was deleted not because it failed as a test but because it was doing policy work.

## Frame-level objection

The candidate accepts the challenger's framing that the split was "fictional" and resolves it by inventing a commit envelope (commit B) for the policy question. That resolution preserves the workflow's self-image (we don't bake policy silently) at the cost of pushing the workflow's manufactured artifact (commit B) onto the operator's worklist.

**The operator did not ask for a policy decision; they asked for an item to be picked up.** The pipeline surfaced a real concern (Done-when #5 was doing hidden work) and converted it into a deliverable on the operator's desk (commit B). That is the workflow generating its own scope. The honest product move is one of:

(a) Apply the 4 edits, delete Done-when #5, write commit message that says nothing about policy, return the policy question to the punch list as a *new line item* (item 12+, not commit B). The operator chose the punch list as the intake surface; respect it.
(b) Apply the 4 edits, delete Done-when #5, name in the commit message that the 7 references remain stale and that the immutability reading is unsettled — and stop there. Let the next item that touches session artifacts force the question.

Both are smaller than commit B. Both honor "the workflow surfaced a concern" without converting concern into deliverable.

## What would change the verdict

Collapse commit B into a punch-list line item (or a single sentence in commit A's message naming the unresolved question), drop the "child of `c2fe604`" framing unless the operator has separately endorsed it, and keep the line-138 parenthetical only if the operator confirms touching the dogfood entry is in scope.

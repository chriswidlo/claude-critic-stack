## Frame as given

## Alternative frame

**The folder-shape migration was a botched rename and item 3 is the cleanup invoice; everything downstream is incidental.** Under this frame, the unit of analysis is not "11 stale references" or "4 edits + 1 policy question," it is `c2fe604` itself — a structural change that landed without its own cleanup pass. Item 3 is the second half of an unfinished commit, not a discrete ticket on a worklist; the optimization target is "make `c2fe604` retroactively atomic," not "advance the punch list" or "settle the immutability rule." Under this frame, the session-artifact occurrences are not a policy question at all — they are leftover work from a migration that should have updated *its own* references in the same commit, and the only reason they feel policy-shaped now is that time has passed and someone wrote a README.md sentence about immutability in the interim. The frame predicts a different shape of fix: a single sweep tied conceptually to `c2fe604`, possibly committed with a message that names that commit as parent, with no policy declaration because there is no policy to declare — only debt.

## Challenge to the split frame itself

The split is **not coherent; it is a deferral dressed as discrimination.** A genuine split would route the two halves to different mechanisms — the mechanical refactor to a small commit, the policy question to a constitutional ticket with its own pipeline pass. The plan as drafted does neither: it ships the mechanical edits *and* declares the policy via Done-when #5 ("No session-artifact file is modified by this commit") without naming the declaration as such. That is the same hidden replace the classifier flagged, simply pushed one frame-revision deeper. The split frame's promise — "we'll handle these halves separately" — has no carrier in the plan. The operator either decides the policy now (in which case it is one frame, not split) or the plan must literally remove Done-when #5 and queue the policy question as a separate ticket (in which case the split is real). The current draft does neither; it claims the split rhetorically while behaving as if it had been collapsed in favor of the strict reading. That is the kind of frame the orchestrator stops interrogating because it sounds careful.

## Challenge to the outside-view's modal failure prediction

The "fence leaks within 6–12 months" prediction is **scary-sounding-but-low-base-rate for this specific repo.** The forecast is anchored on "stated rules in single-operator repos drift" — true in general — but item-3-specific signals push the other direction: (a) this repo already has the immutability rule restated in [`.claude/session-artifacts/README.md`](.claude/session-artifacts/README.md):45-51 *in the directory it governs*, which is exactly the mitigation outside-view §8 recommends as the cheapest lift; the mitigation is partially in place. (b) The "agent helpfully fixes the click-broken link" failure mode requires an agent both to enter a session-artifact directory for an unrelated reason *and* to volunteer a tangential fix; in this stack, agents route through the 12-step workflow and rarely freelance edits. (c) The operator runs an adversarial-review process precisely to catch silent erosion. The outside-view's confidence in the modal failure (~30–40% latent debt, ~20% rule-erosion) treats those structural counter-pressures as absent. The prediction is not unfounded, but it leans on a reference class (small doc repos with paper rules) whose distinguishing feature — no review process — does not match the case at hand. A weaker but better-grounded prediction would be: "the rule will hold under current routing; the failure mode appears only if routing degrades or a new agent class joins that bypasses the workflow."

## Frame-level objection the orchestrator should carry forward

The split frame is fictional — the plan's Done-when #5 quietly ships the strict policy while the frame claims the policy is still open; the generator must either delete Done-when #5 (making the split real) or own the policy declaration explicitly (collapsing the split honestly), but cannot continue to do both.

## Deletion cost (for subsume/replace rows)

- **Default Claude Code system prompt — "surgical changes" coverage** (subsume): Nothing on disk to delete; the default prompt is Anthropic-managed. The deletion is conceptual — the orchestrator stops *relying* on the default's implicit coverage and treats the standing instruction as the load-bearing primitive. Blast radius: zero callers, zero migrations, zero config surface. Cost is purely cognitive: the user must remember the standing instruction is now the source of truth.
- **karpathy-skills plugin packaging mechanism** (replace): Nothing currently installed (`installed_plugins.json` is empty), so no callers, no data migration, no config to remove. The "replacement" is a forward-looking refusal to install rather than an active tear-down. Deletion cost is zero. The cost being paid is *opportunity* cost: the user gives up the auto-update path the marketplace plugin would provide. Stated upside of replacement: explicit version control over the content; no silent overlap-induced drift (outside-view distillation #3); no maintenance asymmetry (#4).

## Requires decision (conflicts)

None of the four principles conflict with an existing primitive in the strict sense (Explore #5 confirmed orthogonality). The one near-conflict is *placement*:

- **karpathy content vs. project CLAUDE.md (claude-critic-stack)**: not a content conflict but a placement one — putting coding-hygiene principles in the deliberation harness's project file would contradict that file's explicit "not a target codebase" disclaimer. The decision is therefore not "which primitive wins" but "which file is the destination." The generator should treat the global `CLAUDE.md` as the only viable destination given the existing primitives; this is not a true conflict requiring user input.

## Preserved primitives with stated reason (non-default)

- **Global agents `master-code-architect`, `code-refactoring-specialist`, `solution-architect`**: preserved because they fire only on explicit invocation (Explore #1–#3) and contain workflow detail beyond the single-line principle (multi-step architectural reasoning, refactoring-specific rules, structured clarification protocols). Standing-instruction karpathy lines do not replace this depth — they cover the always-on baseline. Concrete cost of removal: loss of the deeper agent-mode workflows the user has chosen to keep on disk; no evidence the karpathy content reproduces those workflows.
- **Default Claude Code system prompt — "no comments / no error handling for impossible cases" coverage**: preserved because Anthropic owns it and the user cannot delete it; "extend" is the only honest label. Concrete cost of attempting replacement: would require contradicting the default in user CLAUDE.md, which canon #6 (multi-agent essay: vague vs. explicit) warns against without measurement.

## Primitives the distillations did not name but the query implies

- **Per-repo project CLAUDE.md files in the user's *other* repositories** — Explore explicitly flagged this as a gap (#"did not check whether project-level CLAUDE.md files in other user repos already encode Karpathy-like guidance"). If the user has karpathy-style content in some project files already, the global addition could duplicate or contradict it. Candidate for follow-up Explore if the user names other repos.
- **The Claude Code default system prompt's actual current contents** — canon #5 staleness flag and Explore gap both call out that "defaults cover X" is asserted without inspection. The scope-map treats default coverage as a hypothesis, not a fact; the generator should not rely on default coverage as load-bearing without an empirical check.
- **Token-cost / attention-budget impact of populating a currently-empty global CLAUDE.md** — Explore did not assess; canon #3 (every new token depletes attention budget) makes this load-bearing. The generator should treat additions as having a non-zero standing cost.
- **Whether `quality-assurance-validator` already encodes test-first / verifiable-success discipline** — Explore did not enumerate the agent's contents; if it does, the `extend` label on `quality-assurance-validator` flips to `subsume` and the karpathy "Goal-Driven Execution" principle becomes a duplication rather than a gap-fill. This is the most decision-relevant unknown in the map.
